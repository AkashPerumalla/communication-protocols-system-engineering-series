# Exercise 06 - Telecom Observability Tools

## Objective

Use telemetry, alarm, device, and log tools to query platform operational datasets.

## Steps

1. Invoke telemetry tool with optional `device_id` filter.
2. Invoke alarm tool with optional `severity` filter.
3. Invoke device status and log search tools.

## Solution

Domain services consume seeded sample datasets with deterministic ordering and filter logic.

## Outcome

Operational insights are returned consistently for NOC/SatCom/IoT scenarios.
